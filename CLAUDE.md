## Ключевые правила

### Обязательно перед любым изменением
1. Прочитай этот файл
2. Убедись, что изменение не ломает переключатель RU/EN
3. Проверь адаптивность (mobile ≤ 600px)

### Двуязычность
Все тексты — через атрибуты `data-ru` / `data-en`:
```html
<span data-ru>Текст на русском</span>
<span data-en>Text in English</span>
```
CSS скрывает нужный язык через `body.lang-en [data-ru] { display: none }`.

### Цвета — только через переменные
```css
/* Правильно */
color: var(--gold);

/* Неправильно */
color: #C9A84C;
```

### JavaScript
- Только `const` / `let`, никаких `var`
- Никаких внешних библиотек
- Имена функций — глаголом: `showPreview`, `handleDrop`

### Комментарии в коде
Не писать комментарии, объясняющие что делает код. Только если есть неочевидное «почему».

## Компоненты index.html

| ID | Элемент | Поведение |
|---|---|---|
| `uploadArea` | Зона загрузки | Скрывается после выбора файла |

## Частые задачи

**Добавить поле в форму эксперта**
→ Вставить `<div class="form-group">` в `<div class="form-row">`, добавить оба языка.

**Добавить блок в карточку результата**
→ Скопировать `.info-block` или `.history-block`, добавить оба языка.

**Изменить демо-данные в карточке**
→ Найти `<div class="result-header">` в `index.html`, редактировать напрямую.

**Подключить реальный API**
→ Заменить `setTimeout` в `analyzeBtn` на `fetch('/api/analyze', ...)`, см. `api-guide.md`.
