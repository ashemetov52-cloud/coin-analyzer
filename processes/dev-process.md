# Процесс разработки с Claude Code

## Принцип работы

Проект ведётся через Claude Code (CLI). Вся работа — итерационная: одна задача → один чёткий запрос → проверка результата → следующий шаг. Не стоит делать несколько крупных изменений в одном запросе.

---

## Структура задач

### Приоритеты по уровням

**P0 — Критичные (делаем сразу)**
- Сломана основная функция (загрузка, анализ, форма)
- Визуальный баг виден на первом экране
- API интеграция не работает

**P1 — Важные (делаем в текущей итерации)**
- Новая функция из roadmap
- Улучшение UX на основе обратной связи
- Оптимизация производительности

**P2 — Желаемые (делаем когда есть время)**
- Рефакторинг без изменения поведения
- Дополнительные языки
- Документация

## Как формулировать запросы Claude Code

### Хороший запрос
```
Добавь в карточку результата кнопку "Сохранить PDF".
При нажатии — генерировать PDF с данными карточки через window.print().
Стили для печати уже добавь в <style media="print">.
Кнопка — вторичная (золотая рамка), появляется вместе с карточкой.
```

### Плохой запрос
```
Сделай сайт лучше и добавь больше функций
```

### Шаблон запроса
1. **Что сделать** — конкретное изменение
2. **Где** — файл / компонент / секция
3. **Как должно выглядеть** — ссылка на дизайн-систему или описание
4. **Ограничения** — чего не трогать

## Работа с файлами

### Правило одного файла
Пока проект — это `index.html`, все изменения вносятся только в него. Не создавать отдельные CSS/JS файлы без явной необходимости.

### Перед каждым изменением
1. Прочитай `CLAUDE.md` если не читал в этой сессии
2. Убедись, что изменение не ломает RU/EN переключатель
3. Проверь адаптивность (mobile ≤600px)

### После каждого изменения
1. Открой `index.html` в браузере
2. Проверь: загрузку фото → анализ → карточку результата → форму
3. Переключи язык на EN, убедись что всё переведено

## Соглашения по коду

### HTML
- Все двуязычные тексты — через атрибуты `data-ru` / `data-en`
- Новые секции — с разделителем `.section-divider`
- ID для JS-хуков: `camelCase` (например, `analyzeBtn`, `resultCard`)

### CSS
- Новые цвета только через переменные `var(--token)`
- Не добавлять новые шрифты — только Georgia
- Скругления: `border-radius: 2–4px` максимум
- Анимации: `transition: 0.2s` для hover, именованные `@keyframes` для сложных

### JavaScript
- Никаких внешних библиотек
- Функции именовать глаголом: `showPreview`, `handleDrop`, `submitForm`
- Не использовать `var` — только `const` / `let`

## Частые задачи — быстрые инструкции

### Добавить новое поле в форму эксперта
Вставить `<div class="form-group">` в нужный `<div class="form-row">`, добавить оба языка через `data-ru` / `data-en`.

### Добавить новый блок в карточку результата
Скопировать структуру `.info-block` или `.history-block`, добавить оба языка, при необходимости добавить CSS в секцию `<style>`.

### Добавить новую страницу
Пока страниц нет — добавлять как скрытые секции с `display: none`, показывать через JS.

## Дебаггинг

### Если сломался переключатель языка
Проверить: у каждого текстового элемента есть оба `data-ru` и `data-en`. CSS правило `body.lang-en [data-ru]` скрывает ru-тексты.

### Если не работает drag-and-drop
Убедиться что `e.preventDefault()` есть на `dragover`. Проверить что `fileInput` остаётся поверх upload-area (position: absolute, inset: 0).

### Если карточка результата не появляется
Проверить что `resultCard.classList.add('visible')` вызывается и CSS `.result-card.visible { display: block }` не перекрыт другим правилом.
